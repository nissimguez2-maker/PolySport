# Design Context

Source file for the Impeccable design skill. Synthesized from the current
dashboard templates, CSS tokens, README, and STRATEGY.md — not from user
interview. Update this file as the product evolves.

## Users

Solo operator (one developer), monitoring a pre-trading soccer-arbitrage bot
during the Israel evening trade window (14:00–22:30 IL). This dashboard has
no customers — it is an operator's workbench, in the spirit of Grafana for
a team of one.

Primary job-to-be-done: fast visual triage on a ~15s refresh cadence.
- Is the logger polling, and how stale is the last snapshot?
- Did the Pinnacle → Polymarket matcher silently drop any pairs?
- What is the best divergence right now, and does any match pass all entry
  gates (div ≥ 2¢, spread ≤ 3¢, depth ≥ $500, fav prob < 0.80)?
- How much of the 20,000-req/mo Odds API quota has been burned?

Phase 1 is logging only — no live trading. The bar for the dashboard is
"nothing unhealthy can hide in plain sight," not "user feels delight."

## Brand Personality

Three words: **operator-serious, dense, green-as-signal.**

- **Operator-serious** — the tool is one phase away from touching real money.
  No playful microcopy, no onboarding niceties, no friendly errors. The
  error state is `⚠ Dashboard fetch failed` in red, because that's what an
  operator needs to see.
- **Dense** — closer to a Bloomberg row than a marketing card. ~10.5px
  uppercase labels, tabular-nums numbers, sub-1rem tile padding, the KPI
  strip and match table fit on one screen without scrolling.
- **Green-as-signal** — green is the color of healthy state, positive
  divergence, and the BUY signal. It is an accent and a meaning, not a
  brand tint. Earlier versions tinted every neutral toward pitch-green;
  that worked in dark mode but broke in light mode, so neutrals are now
  warm-gray on both themes and green appears only where it carries
  operational meaning.

Emotional goal: calm when healthy, immediate alarm when broken. The
breathing pulse dot is the only ambient motion; red and amber only appear
on anomalies.

## Aesthetic Direction

**Theme: hybrid, OS-driven.** `color-scheme: light dark` with
`light-dark()` tokens so the dashboard auto-adapts to the operator's
preferred-color-scheme. An evening desk may run dark; a daytime browser
check may run light. Both work.

**Palette: two parallel token stacks, one per theme.**
- Light neutrals: `--bg: #fafaf7`, `--surface: #ffffff`,
  `--surface2: #f0f1ec`, `--border: #e3e5df`, `--muted: #6a6f68`,
  `--text: #1a1e1b`, `--text-bright: #0a0d0b`. Warm-gray, untinted —
  tinting off-white toward green reads seasick in practice.
- Dark neutrals (unchanged from the prior palette): `--bg: #0e1210`,
  `--surface: #141a15`, `--surface2: #1a2219`, `--border: #232e24`,
  `--muted: #7a8c7c`, `--text: #e8ede9`. These still tint toward the
  brand green because subtle green-on-near-black reads cleanly.
- Semantic accents pick darker values on light and brighter on dark so
  text always clears 4.5:1: `--green` is `#1f7a3d` / `#4eba6f`,
  `--red` is `#a81a1a` / `#e05252`, `--amber` is `#7d5b00` / `#d4a418`.
- Tints and borders derive from the active semantic via
  `color-mix(in srgb, var(--green) N%, transparent)` so one rule covers
  both themes. Raw `rgba()` is limited to the six-league identity
  sub-palette.
- No cyan, no neon, no gradients. Ever.

The six league pills (EPL blue, UCL purple, L1 orange, SA green, LL red,
BL amber) are a distinct sub-palette — they're identity tags, not semantic
signals. Do not repurpose these hues elsewhere in the UI.

**Typography: DM Sans (UI) + DM Mono (numbers).**

DM Sans appears on Impeccable's reflex-font-reject list. **Its use here is
deliberate and load-bearing** — do NOT swap it out in a future audit pass.
The Sans/Mono pairing is the point:
- DM Mono with `font-variant-numeric: tabular-nums` carries every number
  in the dashboard so columns line up digit-for-digit and a changed digit
  is caught on a 15s scan.
- DM Sans carries labels, team names, and signal chips.
- OpenType feature `font-feature-settings: "cv11" 1` enables the alt zero.
- Weights: DM Sans 300/400/500/600, DM Mono 400/500.

**Motion: restrained, functional, ease-out only.**
- 150ms `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-expo) as the house
  transition timing.
- `flash-update` on HTMX swap is the same curve over 300ms (one pulse,
  not continuous).
- `.pulse` dot uses 2.4s ease-in-out for a breathing rhythm on the Logger
  tile — the one ambient motion in the app.
- No bounce, no elastic, no spring.

**References:** Grafana dashboards, Datadog observability tiles, Bloomberg
Terminal row density. Light-theme lineage: Linear, GitHub, Stripe — crisp
off-white with semantic color kept sparing.

**Anti-references (do not drift toward these):**
- Consumer-SaaS marketing pages: hero metric, gradient accent, generous
  whitespace, card grids of identical tiles.
- "Dark mode with cyan glow" AI-slop palette — purple-to-blue gradients,
  neon accents on black.
- Light-mode-with-pastels: pastel greens/pinks don't carry alarm weight;
  semantic colors must stay saturated enough to mean something.
- Playful empty states, illustrated error states, friendly microcopy.
- Side-stripe borders >1px as a colored accent (forbidden by Impeccable
  BAN 1). The codebase's intentional escape pattern is
  `box-shadow: inset 3px 0 0 var(--green)` on the `.buy` match row — do
  not "correct" this to a border-left.
- Gradient text of any kind (Impeccable BAN 2). Solid colors only.

## Design Principles

1. **Color is semantic, never decorative.** Green = healthy / opportunity.
   Red = anomaly / silent failure. Amber = approaching trade window /
   caution. League-pill hues are identity tags, not signal colors. If you
   introduce a new red, it must be something an operator should act on —
   never dilute the alarm palette with decorative reds.

2. **Tabular monospace for every number, sans-serif for every label.**
   The dashboard is scanned, not read. Numbers must align vertically so
   the eye catches a changed digit without re-reading. `font-variant-
   numeric: tabular-nums` is set globally — preserve it.

3. **Density over whitespace.** Sub-1rem tile padding, ~10.5px uppercase
   labels, a two-tier KPI layout (three primary tiles at ~34px values
   over a six-cell health strip), and a single 8-column match-table card.
   Spacing exists to create rhythm, not to "breathe." Do not space this
   out like a marketing page.

4. **Hierarchy reflects operational importance.** Tier 1 are the three
   numbers an operator stares at (Would fire now, Best gap now, In trade
   window). Tier 2 is system health — glance, don't dwell. Never give
   Total Fixtures the same visual weight as Would-Fire.

5. **Silence when healthy, alarm when broken.** Default state is calm:
   bright numbers on neutral surfaces, one breathing pulse dot, no
   ambient color. Any red or amber on the screen means something happened.
   Do not add decorative motion, glow, or color that would train the eye
   to ignore the alarms.

6. **One grid rhythm, shared primitives.** Tier-1 and Tier-2 KPIs reuse
   the same `.kpi` tile primitive with container-driven density overrides.
   `.league`, `.t-pill`, `.age-chip`, and `.chip` (BUY / SELL / SKIP) each
   reuse one shape across every surface they appear on. Fewer primitives =
   fewer decisions to second-guess in review.
